## 1. ROLE IN THE PROJECT

Logistic regression serves as the baseline model. It assumes a linear
relationship between features and the log-odds of Monday opening higher
than the previous Friday's close (GapUp = 1). Its value in this project
is twofold:
  - It provides a performance floor that XGBoost and LSTM must beat.
  - Its coefficients are directly interpretable: we can state exactly
    which features push the prediction toward gap-up or gap-down, and
    by how much. This is not possible with the other two models.


## 2. DATASET

Source file: structured_csv_data_files/fetched_data/dataset_clean.csv
Produced by: notebooks/eda.ipynb

  Total rows after cleaning:    8,437
  Primary set (no COVID):       8,146 rows   GapUp rate = 51.6%
  Extreme-event set (COVID):      291 rows   GapUp rate = 45.0%

The dataset covers 25 large-cap US software companies
(Software-Application and Software-Infrastructure per SEC classification)
from April 2016 to December 2024. Each row represents one Monday.

Tickers: ADBE, ADP, ADSK, AKAM, CDNS, CPAY, CRM, FFIV, FICO, FTNT,
         GEN, GWRE, HUBS, INTU, NOW, NTAP, PANW, PAYX, PTC, ROP,
         SNPS, SSNC, TYL, VRSN, WDAY


## 3. TARGET VARIABLE

  GapUp = 1  if Monday Open > previous Friday Close  (gap up)
  GapUp = 0  if Monday Open <= previous Friday Close (gap down)

  Gap magnitude = (Monday Open - Friday Close) / Friday Close
  Median gap magnitude: +0.017%   Std: +/-1.22%

The gap is typically very small in absolute terms. The distribution is
approximately symmetric around zero, making this a near-balanced
binary classification problem (~51/49 split).


## 4. FEATURE SET — 18 FEATURES

Started with 25 features in the cleaned dataset. Reduced to 18 through
two sequential steps: VIF pruning (statistical analysis notebook) and
leakage removal (identified during EDA review).

### 4a. Features dropped

  VIF pruning (collinearity too high for stable LogReg coefficients):
    RSI           — VIF = 27   collinear with StochPercK and MFI
    GrossMargin   — VIF = 1035 algebraically derived from Revenue and
                               GrossProfit, near-perfectly collinear
                               with NetMargin
    WeeklyRange   — VIF = 10.9 collinear with ATR and BollingerBandWidth

  Look-ahead leakage (same-week Friday data stamped onto Monday rows):
    WeeklyReturn       — (FridayClose / MondayOpen) - 1 for SAME week
    FridayPosition     — where same-week Friday closed in week's range
    OpenCloseSpread    — same-week Friday open-close spread
    IntraWeekVolatility — std of daily returns including same-week Friday

  All four leaky features were computed in the pipeline using
  .groupby([Ticker, week_period]).transform(), which broadcasts the
  Friday value to every row in that week including Monday. On Monday
  morning, Friday has not yet happened — these values do not exist.

### 4b. Final 18 features

  MOMENTUM (3):
    MACD        — trend momentum via EMA(12) - EMA(26) crossover
    ROC         — rate of price change over 12 periods
    StochPercK  — momentum relative to recent high-low range
    [RSI dropped due to VIF]

  TREND (3):
    CloseVEma50 — close price relative to 50-day EMA (medium-term bias)
    CloseVSma20 — close price relative to 20-day SMA (short-term bias)
    ADX         — strength of trend regardless of direction

  VOLATILITY (3):
    BollingerBandWidth — width of Bollinger Bands (volatility expansion)
    ATR                — Average True Range (absolute volatility level)
    FiveDStdDev        — 5-day rolling std of close (reactive volatility)

  VOLUME (3):
    OBV         — On-Balance Volume (directional volume pressure)
    MFI         — Money Flow Index (price x volume combined)
    VolumeRatio — Friday volume vs Mon-Thu weekly average

  FUNDAMENTAL (3):
    NetMargin    — net income / revenue (company profitability)
    RoA          — return on assets (balance sheet efficiency)
    RevGrowthQoQ — quarter-over-quarter revenue growth rate

  MISSINGNESS INDICATORS (3):
    GrossMargin_missing     — 1 for CDNS, WDAY (don't file COGS)
    CostOfRevenues_missing  — 1 for CDNS, WDAY
    Liabilities_missing     — 1 for ADSK, CPAY (different XBRL tags)

  Note on missingness indicators: rather than dropping tickers with
  missing fundamental data, binary flags were added. This preserves
  all rows and lets the model learn that the absence of COGS reporting
  is itself informative (typically signals a pure-software margin
  structure). The actual feature values (NetMargin, RoA) were available
  for all tickers.


## 5. PREPROCESSING PIPELINE

Logistic regression and its L1 penalty are scale-sensitive. Unscaled
features cause the optimizer to be dominated by large-magnitude
variables and the penalty to be applied unevenly. Three treatments:

  OBV — per-ticker z-score:
    OBV is a cumulative quantity. Its absolute level is arbitrary,
    depending on when the ticker started trading. Z-scoring within each
    ticker (mean/std computed from training data only) removes the level
    and preserves the directional signal.

  ATR, BollingerBandWidth, FiveDStdDev, VolumeRatio — log1p then StandardScaler:
    These are right-skewed with occasional large spikes during high-
    volatility periods. Log-compressing before scaling prevents extreme
    values from disproportionately influencing coefficient estimates.

  All other continuous features — StandardScaler only.

  Missingness indicators — no scaling (already binary 0/1).

CRITICAL: the scaler is fit on the training fold only and applied to
both train and test within each fold. Fitting on the full dataset would
leak future distribution information into the training window.


## 6. REGULARIZATION

  Model: LogisticRegression(penalty='l1', solver='liblinear')

  L1 (Lasso) regularization was chosen over L2 (Ridge) for three reasons:
    1. Small effect sizes — StochPercK had the strongest Mann-Whitney
       effect size at r=0.19 (small-to-medium). With weak signals and
       18 features, unregularized logistic regression would overfit.
    2. Residual collinearity — even after VIF pruning, moderate
       correlations remain within the momentum and volatility clusters.
       L1 zeroes out one of each correlated pair.
    3. Implicit feature selection — L1 produces sparse solutions,
       making the final model more interpretable for the paper.

  Regularization strength C (inverse of penalty):
    Smaller C = stronger regularization (more coefficients zeroed out)
    Grid searched: [0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1.0, 5.0, 10.0]
    Best C was consistently small (0.005-0.05), confirming strong
    regularization is appropriate for this dataset.


## 7. WALK-FORWARD VALIDATION

  Why walk-forward and not k-fold cross-validation:
    - Features are non-stationary (OBV, ATR drift over time)
    - The target's base rate shifts across years (gap-up rate differs
      systematically between bull and bear market years)
    - Random k-fold would mix future data into training folds,
      producing unrealistically optimistic results

  Structure — expanding window, one year per fold:
    Fold 1: Train 2016-2018 (2,682 rows) -> Test 2019 (992 rows)
    Fold 2: Train 2016-2019 (3,674 rows) -> Test 2020 (613 rows)  *COVID
    Fold 3: Train 2016-2020 (4,287 rows) -> Test 2021 (917 rows)
    Fold 4: Train 2016-2021 (5,204 rows) -> Test 2022 (997 rows)
    Fold 5: Train 2016-2022 (6,201 rows) -> Test 2023 (953 rows)
    Fold 6: Train 2016-2023 (7,154 rows) -> Test 2024 (992 rows)

  C tuning per fold:
    To avoid using the test year for hyperparameter selection, the last
    year of each training window is held out as an inner validation set.
    Best C is selected by AUC on this inner set, then the model is
    refit on the full training window before evaluating on the test year.

## 9. COEFFICIENT ANALYSIS

L1 regularization zeroed out several features entirely. The table below
shows average coefficient and how many of the 6 folds each feature was
active (non-zero):

  Feature                | Avg coef | Avg |coef| | Folds active
  -----------------------|----------|-------------|-------------
  StochPercK             | +0.2519  | 0.2519      | 6/6  <- most stable
  MFI                    | +0.0558  | 0.0558      | 4/6
  MACD                   | -0.0462  | 0.0462      | 4/6
  BollingerBandWidth     | -0.0308  | 0.0308      | 3/6
  ROC                    | +0.0032  | 0.0180      | 3/6
  VolumeRatio            | -0.0152  | 0.0152      | 3/6
  OBV                    | +0.0086  | 0.0143      | 2/6
  NetMargin              | +0.0107  | 0.0107      | 3/6
  ATR                    | -0.0083  | 0.0083      | 2/6
  ADX                    | -0.0055  | 0.0055      | 2/6
  FiveDStdDev            | +0.0029  | 0.0029      | 2/6
  CloseVEma50            | -0.0015  | 0.0015      | 2/6
  RoA                    | +0.0012  | 0.0012      | 1/6
  RevGrowthQoQ           | -0.0001  | 0.0007      | 2/6
  CloseVSma20            |  0.0000  | 0.0000      | 0/6  <- zeroed out
  GrossMargin_missing    |  0.0000  | 0.0000      | 0/6  <- zeroed out
  CostOfRevenues_missing |  0.0000  | 0.0000      | 0/6  <- zeroed out
  Liabilities_missing    |  0.0000  | 0.0000      | 0/6  <- zeroed out

Key interpretations:

  StochPercK (+0.25, 6/6 folds):
    By far the most stable and influential feature. Positive coefficient
    means higher StochPercK (stock closing near the top of its recent
    high-low range on Friday) predicts a gap up on Monday. Intuition: a
    stock finishing the week with strong upward momentum tends to open
    the following Monday continuing that directional bias.

  MACD (-0.046, 4/6 folds):
    Negative coefficient. A higher MACD (bullish crossover) on Friday
    associates with gap DOWN on Monday. This is counterintuitive and
    may reflect a mean-reversion dynamic: stocks that have already
    moved strongly upward during the week give back some gains at
    Monday's open.

  MFI (+0.056, 4/6 folds):
    Positive. Higher Money Flow Index (buying pressure combined with
    volume) on Friday associates with gap up Monday. Consistent with
    the idea that institutional accumulation late in the week carries
    into the next week's open.

  BollingerBandWidth (-0.031, 3/6 folds):
    Negative. Wider bands (higher recent volatility) associate with
    gap down. In high-volatility regimes, the market is uncertain and
    tends to open lower Monday as risk-off sentiment dominates.

  Missingness indicators (all 0/6 folds):
    CloseVSma20 and all three missingness indicators were regularised
    out in every fold. The model found no linear predictive value in
    whether a ticker reports COGS or not, once other features present.


## 10. AIC / BIC MODEL COMPARISON

Computed using statsmodels.Logit on the final fold (train 2016-2023,
test 2024). Lower is better. BIC penalises additional features more
heavily than AIC at large n.

  Model                       | Features | AIC    | BIC    | Log-Lik
  ----------------------------|----------|--------|--------|----------
  Full model (18 features)    |    18    |  N/A   |  N/A   | singular matrix
  L1-selected (13 features)   |    13    | 9687.7 | 9783.9 | -4829.8
  Technical only (12 features)|    12    | 9686.6 | 9776.0 | -4830.3

  The full 18-feature model produced a singular matrix — confirming
  that residual collinearity among the 5 zeroed-out features makes
  unregularized estimation infeasible without L1.

  The L1-selected model and technical-only model have near-identical
  AIC/BIC, with technical-only marginally better on BIC. This suggests
  that in the linear modelling context, the fundamental features
  (NetMargin, RoA, RevGrowthQoQ) add minimal incremental information
  beyond what technical indicators already capture.


## 11. SECONDARY EVALUATION — COVID CRASH (FEB-MAY 2020)

The 291 COVID-period rows were held out from all training and primary
evaluation. As a robustness test, we took the model trained on 2016-2019
(the fold immediately preceding COVID) and evaluated it on these rows.

  COVID period GapUp rate:  45.0%  (more gap-downs than normal)
  Majority-class baseline:  55.0%  (always predict gap-down)

  Secondary results:
    Accuracy:  0.684
    Precision: 0.638
    Recall:    0.687
    F1:        0.662
    AUC-ROC:   0.781

  Comparison:
    Normal-market avg AUC:  0.574
    COVID-period AUC:       0.781
    Apparent improvement:  +0.207

  IMPORTANT INTERPRETATION:
    The higher AUC during the COVID crash does NOT mean the model is
    robust to extreme market conditions. The COVID crash was a sustained
    directional event — nearly every Monday was a large gap-down for
    weeks on end. The momentum features (MACD, MFI, StochPercK) were
    deeply negative going into those Mondays, and the model had learned
    during normal markets that deeply negative momentum predicts gap-down.
    That signal happened to be directionally correct during the crash,
    but at scales far beyond anything in the training data.

    If the crash had involved chaotic up-and-down swings rather than a
    sustained fall, model performance would likely have been poor. Report
    this in the paper with the caveat: "The model's strong secondary
    performance on the COVID period reflects the unidirectional nature
    of that crash, not genuine robustness to extreme market conditions."


## 12. KEY LIMITATIONS

  1. Linear assumption: logistic regression cannot capture non-linear
     relationships between features and the target. The mutual information
     analysis showed NetMargin has non-linear signal (high MI, low MW
     effect size) that LogReg cannot exploit. This is a structural
     limitation — XGBoost is expected to outperform on this dimension.

  2. No inter-ticker relationships: the model treats each (ticker, week)
     observation independently. It cannot learn sector-wide signals, such
     as whether large-cap software stocks collectively had a strong Friday.

  3. Leaky price aggregates removed: the four price aggregate features
     (WeeklyReturn, FridayPosition, OpenCloseSpread, IntraWeekVolatility)
     were removed due to look-ahead leakage. Replacing them with prior-
     week equivalents is a potential improvement for future work.

  4. Stationarity of OBV and ATR: despite per-ticker z-scoring, OBV
     remains non-stationary in most tickers. This is partially mitigated
     by the walk-forward structure but is a known limitation.
