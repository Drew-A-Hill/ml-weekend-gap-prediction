--------------------------------------------------------------------------------
1. ROLE OF LSTM IN THE PROJECT
--------------------------------------------------------------------------------
LSTM is the third model and serves as the sequential pattern detector. Logistic
regression and XGBoost treat each Monday row as an independent observation —
they cannot see whether RSI has been rising for three consecutive weeks, or
whether volume has been steadily building. LSTM (Long Short-Term Memory) is
designed specifically to learn from sequences: it processes 4 consecutive weekly
observations per ticker and decides what information to carry forward (the memory
cell) and what to discard (the forget gate).

The key research question LSTM answers:
  Is there sequential, week-over-week signal in technical features that static
  models cannot detect?

Answer: No — LSTM does not outperform static models on this task. This is a
substantive finding, not a failure, and is consistent with the broader literature
on tabular financial data.

--------------------------------------------------------------------------------
2. CONFIGURATION — CHOSEN VIA SYSTEMATIC EXPERIMENT
--------------------------------------------------------------------------------
Before finalising the notebook, six variants were tested across three axes:

  Axis 1: Sequence window — 4 weeks vs 8 weeks
  Axis 2: Training strategy — expanding window vs rolling 3yr vs rolling 4yr
  Axis 3: Learning rate grid — {0.001} vs {0.001, 0.0005}

Results (avg AUC across non-COVID folds):
  Baseline  (win=8, expanding, lr=0.001)      0.5389
  Rolling3  (win=8, roll=3yr,  lr=0.001)      0.5490
  Rolling4  (win=8, roll=4yr,  lr=0.001)      0.5502
  ShortWin  (win=4, expanding, lr=0.001)      0.5548
  Short+R3  (win=4, roll=3yr,  lr=0.001)      0.5614  <- best
  WiderLR   (win=8, expanding, lr=0.001/5e-4) 0.5389

Short window (4 weeks) + rolling 3-year training was the clear winner.
The wider learning rate grid produced identical results to the baseline,
confirming lr=0.001 is already optimal.

Final configuration: window=4, rolling 3-year training window.

--------------------------------------------------------------------------------
4. FEATURE SET — 14 TECHNICAL FEATURES (NO FUNDAMENTALS)
--------------------------------------------------------------------------------
LSTM uses fewer features than XGBoost (14 vs 21). Fundamentals and missingness
indicators are excluded.

  Momentum (4):    RSI, MACD, ROC, StochPercK
  Trend (3):       CloseVEma50, CloseVSma20, ADX
  Volatility (3):  BollingerBandWidth, ATR, FiveDStdDev
  Volume (3):      OBV, MFI, VolumeRatio
  Price Agg (1):   WeeklyRange

Why no fundamentals:
  GrossMargin, NetMargin, RoA, RevGrowthQoQ are reported quarterly. In the
  dataset, each value repeats across ~13 consecutive weekly rows. Feeding a flat
  dimension (same value for 4 consecutive timesteps) gives the LSTM gate
  mechanism nothing temporal to learn. The model would treat it as a static
  bias — exactly what LogReg and XGBoost already do, with no benefit from
  sequencing. Missingness indicators are excluded for the same reason.

Why no leaky features:
  Same 4 exclusions as all models: WeeklyReturn, FridayPosition,
  OpenCloseSpread, IntraWeekVolatility (look-ahead leakage confirmed via code
  inspection of price_aggregate.py — .transform() broadcasts same-week Friday
  data to Monday rows).

--------------------------------------------------------------------------------
5. WHY THE 4-WEEK WINDOW
--------------------------------------------------------------------------------
8 weeks was the initial choice. Systematic experiments showed 4 weeks
outperformed on average. The likely reason:

  - Short windows focus on near-term momentum (~1 month), which is more
    consistent across market regimes.
  - 8-week patterns are more regime-specific: a momentum pattern that persisted
    for 8 weeks in 2017-2019 may not repeat in 2022-2024's different macro
    environment (rate hikes, AI rally).
  - Shorter sequences also produce more training examples (8,071 vs 7,971)
    with lower variance per sequence.

--------------------------------------------------------------------------------
6. WHY THE ROLLING 3-YEAR TRAINING WINDOW
--------------------------------------------------------------------------------
Standard walk-forward uses an expanding window: fold 2024 trains on 2016-2023.
This means 7 years of potentially stale data dominate the training set.

The rolling window discards data older than 3 years:
  - Fold 2024: trains on 2021-2023 only (not 2016-2023)
  - Prevents 2016-2018 pre-AI-era market dynamics from contaminating models
    that will predict 2022-2024 behaviour
  - Consistent training set size per fold (~2,500-2,900 sequences) instead of
    an ever-growing window that inflates older market regimes

This is particularly important for LSTM because sequential patterns are
regime-specific. When the tech sector dynamics shift (e.g., 2022 rate-hike
selloff, 2023-2024 AI rally), historical weekly momentum sequences become
misleading rather than informative.

Rolling window does not apply to LogReg or XGBoost — static models read current
feature values, not trajectories, so older data adds statistical coverage rather
than regime noise.

--------------------------------------------------------------------------------
7. PREPROCESSING — REQUIRED FOR LSTM
--------------------------------------------------------------------------------
Unlike XGBoost, LSTM is sensitive to feature scale. The LSTM cell uses sigmoid
and tanh activations internally — large input values saturate these functions,
collapsing gradient flow during backpropagation (vanishing gradient at the
input level).

Approach: StandardScaler fitted on all training sequences (all timesteps
flattened to 2D), then applied to validation and test sequences.

  - Fit on training data only — no leakage into validation/test
  - Scaler is re-fitted for each fold using only that fold's training years
  - OBV: the global StandardScaler effectively z-scores it per fold

Contrast with XGBoost (no scaling needed) and LogReg (same scaling approach,
but with additional per-ticker OBV z-score and log1p transforms).

--------------------------------------------------------------------------------
8. LSTM ARCHITECTURE
--------------------------------------------------------------------------------
  Type:          Single-layer LSTM
  Input:         (batch, seq_len=4, input_size=14)
  LSTM layer:    hidden_size tuned (32 or 64), num_layers=1
  Dropout:       Applied to last-timestep output before linear layer
  Output head:   Linear(hidden_size, 1) -> Sigmoid -> probability of GapUp
  Loss:          Binary Cross-Entropy (BCELoss)
  Optimizer:     Adam, lr=0.001
  Early stopping: patience=10 epochs on inner val AUC

Why single layer:
  Multi-layer LSTMs require more data to train effectively. With ~2,500-2,900
  training sequences per fold, a second layer would overfit without improving
  generalisation.

Why last-timestep output:
  The final hidden state h_t encodes all prior sequence information via the
  LSTM's memory cell.

14. WHY LSTM DOES NOT OUTPERFORM STATIC MODELS
--------------------------------------------------------------------------------
Three mutually reinforcing reasons:

  1. Features already encode temporal dynamics
     RSI is a 14-period rolling average of gains and losses. MACD is a
     difference of two EMAs. BollingerBandWidth is a rolling standard deviation.
     These features distil multi-week momentum into a single current value. When
     the LSTM receives sequences of these features, most temporal signal has
     already been compressed. The LSTM finds diminishing returns from sequencing
     data that is already temporally encoded.

  2. Data scarcity for a sequence model
     The largest training fold has ~2,900 sequences. LSTMs excel when trained
     on millions of examples with tight autocorrelation at high frequency (NLP,
     speech, sensor streams). At weekly frequency with ~2,900 sequences and
     loose autocorrelation, the architecture cannot reliably learn generalizable
     sequential patterns.

  3. Low target autocorrelation
     The ACF/PACF analysis in the statistical analysis notebook found only 5 of
     25 tickers show significant autocorrelation in the GapUp target. This is
     direct evidence that sequential gap patterns are rare and ticker-specific —
     not the kind of strong, widespread temporal structure that LSTM is designed
     to exploit.

This finding is consistent with Grinsztajn et al. (2022) — "Why tree-based
models still outperform deep learning on tabular data" (NeurIPS) — which showed
gradient boosted trees match or outperform deep learning on structured datasets,
particularly at small-to-medium scale.

15. SECONDARY EVALUATION — COVID EXTREME EVENTS
--------------------------------------------------------------------------------
Setup: Train on 2016-2019 (pre-COVID), test on 291 COVID rows (Feb-May 2020).

  Metric      LSTM    XGBoost   LogReg
  ------      ----    -------   ------
  Accuracy    0.546   0.639     —

  AUC-ROC     0.541   0.724     0.781
  Baseline    0.550   0.550     —

LSTM achieves the lowest COVID AUC of all three models (0.541).

Interpretation:
  The COVID crash is a severe regime shift — extreme feature values, directional
  cascade. LSTM's sequential patterns learned from 2016-2019 normal markets are
  the most disrupted. LogReg's simple linear boundary generalises best under
  distribution shift because it makes fewer assumptions about the decision region
  shape.

  Same caveat as other models: elevated COVID AUC is an artifact of the crash
  being directionally unidirectional, not evidence of robustness.

16. OVERALL INTERPRETATION FOR THE PAPER
--------------------------------------------------------------------------------
The three-model narrative:

  "Logistic regression establishes a linear signal baseline (AUC ~0.574).
  XGBoost confirms that non-linear feature interactions add no additional signal
  (2-7 trees selected by early stopping, AUC 0.566). LSTM confirms that
  sequential week-over-week dynamics add no signal beyond what the engineered
  features already encode (AUC 0.557). All three models modestly outperform
  random (0.500), consistent with the semi-strong form of market efficiency:
  a small, real, but marginally exploitable signal exists in publicly available
  technical and fundamental features. Model complexity does not help — the
  simplest model wins."

Three findings for the paper:

  1. Sequential patterns do not add predictive power over static features.
     The gap prediction signal is captured in the current week's feature values,
     not in how those values evolved over the prior 4 weeks.

  2. Model complexity does not help; simplicity is an advantage.
     LogReg > XGBoost > LSTM on average AUC. Under near-efficient market
     conditions with weak signal, more complex models overfit to noise.

  3. Regime sensitivity increases with model complexity.
     LogReg is most robust across 2019-2024. XGBoost degrades in 2021 (too few
     trees). LSTM degrades in 2022 and 2024 (regime-specific sequential patterns
     don't transfer). Simpler decision boundaries generalise better when the
     data-generating process is non-stationary.

18. LIMITATIONS
--------------------------------------------------------------------------------
  1. Architecture not exhaustively tuned: GRU, attention, or transformer-based
     models were not tested. The literature does not strongly suggest they would
     outperform at this dataset size, but they remain untested.

  2. Features already encode temporal information: results cannot be generalised
     to raw OHLCV input. An LSTM fed raw price/volume sequences without
     engineered features might find different signal.

  3. Rolling window removes historical data: the 3-year rolling strategy
     discards pre-2021 data for fold 2024. The trade-off (recency vs volume)
     is problem-specific and was selected empirically.

  4. Pooled model: one LSTM trained across all 25 tickers. Ticker-specific
     sequential patterns are not modelled.

  5. Weekly frequency: LSTM literature largely validates performance at higher
     frequencies (intraday, daily). Weekly frequency with 4-week windows is
     arguably too low frequency for the architecture to demonstrate its
     strengths.
