--------------------------------------------------------------------------------
1. ROLE OF XGBOOST IN THE PROJECT
--------------------------------------------------------------------------------
XGBoost is the second of three models and serves as the non-linear ensemble
baseline. Where logistic regression assumes a linear decision boundary, XGBoost
builds an ensemble of decision trees where each tree corrects the residual errors
of the previous one (gradient boosting). This allows it to:
  - Capture non-linear relationships between features and the target
  - Automatically model feature interactions (e.g., RSI high AND volume spike)
  - Handle features at different scales without normalization

The key question XGBoost answers: is there non-linear or interaction-based signal
in our feature set that logistic regression cannot exploit?

Answer: No. XGBoost stops at 2-7 trees on most folds and slightly underperforms
logistic regression on AUC. This is a substantive finding, not a failure.

--------------------------------------------------------------------------------
2. DATASET
--------------------------------------------------------------------------------
  - Source:    structured_csv_data_files/fetched_data/dataset_clean.csv
  - Primary:   8,146 Monday rows (GapUp rate: 51.6%)
  - Extreme:   291 COVID rows (Feb-May 2020, GapUp rate: 45.0%) — held out
  - Period:    2016–2024, 25 large-cap US software companies
  - Target:    GapUp (1 = Monday Open > previous Friday Close, 0 = otherwise)

--------------------------------------------------------------------------------
3. FEATURE SET — 21 FEATURES
--------------------------------------------------------------------------------
XGBoost uses 21 features vs 18 for logistic regression. Three features dropped
from LogReg due to high VIF are re-introduced here because tree-based models are
immune to multicollinearity: each split evaluates one feature at a time, so
collinear features simply compete for the same split — no coefficient instability.

  Momentum (4):    RSI*, MACD, ROC, StochPercK
  Trend (3):       CloseVEma50, CloseVSma20, ADX
  Volatility (3):  BollingerBandWidth, ATR, FiveDStdDev
  Volume (3):      OBV, MFI, VolumeRatio
  Fundamental (4): GrossMargin*, NetMargin, RoA, RevGrowthQoQ
  Price Agg (1):   WeeklyRange*
  Missingness (3): GrossMargin_missing, CostOfRevenues_missing, Liabilities_missing

  * = re-added vs LogReg (were VIF-pruned in LogReg)

Still excluded from all models (confirmed look-ahead leakage):
  WeeklyReturn, FridayPosition, OpenCloseSpread, IntraWeekVolatility
  These use same-week Friday data broadcast to Monday rows via .transform(),
  which does not exist at prediction time Monday morning.

No preprocessing or scaling is applied. XGBoost is invariant to feature scale.

--------------------------------------------------------------------------------
4. NO PREPROCESSING REQUIRED
--------------------------------------------------------------------------------
Decision tree splits are threshold-based: "if RSI > 62.3, go left." The actual
scale of the feature does not matter — only the relative ordering. Therefore:
  - No standardization or z-scoring
  - No log transforms
  - No per-ticker normalization
  - All 21 features fed raw

This is a meaningful difference from logistic regression, which required
per-ticker z-scoring for OBV and log1p transforms for skewed features.

--------------------------------------------------------------------------------
5. VALIDATION STRATEGY — WALK-FORWARD (EXPANDING WINDOW)
--------------------------------------------------------------------------------
Identical structure to logistic regression for direct comparison.

  Fold 1: Train 2016-2018 (2,682 rows) → Test 2019 (992 rows)
  Fold 2: Train 2016-2019 (3,674 rows) → Test 2020 (613 rows)  ← COVID fold
  Fold 3: Train 2016-2020 (4,287 rows) → Test 2021 (917 rows)
  Fold 4: Train 2016-2021 (5,204 rows) → Test 2022 (997 rows)
  Fold 5: Train 2016-2022 (6,201 rows) → Test 2023 (953 rows)
  Fold 6: Train 2016-2023 (7,154 rows) → Test 2024 (992 rows)

Temporal integrity maintained throughout — no shuffling, no leakage across folds.
The COVID fold (2020) is included in training data for subsequent folds but
excluded from primary metric averaging due to non-stationarity of that period.

Hyperparameter tuning (inner validation):
  The last calendar year of each training window is carved off as an inner
  validation set for hyperparameter selection. Best params are selected by
  inner val AUC, then the model is refit on the full training window.

--------------------------------------------------------------------------------
6. HYPERPARAMETER GRID — 48 COMBINATIONS
--------------------------------------------------------------------------------
  max_depth:         [3, 4, 5]         — tree complexity
  learning_rate:     [0.05, 0.1]       — shrinkage per tree
  subsample:         [0.7, 0.9]        — row sampling fraction per tree
  colsample_bytree:  [0.7, 0.9]        — feature sampling fraction per tree
  min_child_weight:  [1, 3]            — minimum leaf node weight

  n_estimators = 500 with early_stopping_rounds = 30
  (tree count auto-determined by inner validation AUC plateau)

Early stopping means: if inner validation AUC does not improve for 30 consecutive
trees, training stops. This prevents overfitting and determines n_estimators
automatically per fold.

--------------------------------------------------------------------------------
8. THE FEW-TREES FINDING (KEY RESULT)
--------------------------------------------------------------------------------
Most folds stopped at 2-7 trees (folds 2021: 2 trees, 2023: 3 trees, 2020: 7
trees). This is the central diagnostic finding of the XGBoost analysis.

Validation check performed: re-ran all 6 folds with fixed n_estimators=200
(no early stopping), using the same best hyperparameters from the grid search.

Result: AUC delta was negative or near zero across folds. More trees did not help;
in most cases they slightly hurt (mild overfitting).

Interpretation:
  XGBoost exhausts all learnable non-linear structure in 2-7 splits. The model
  is not stopping prematurely due to a noisy inner validation set — it genuinely
  finds no more signal. This has two implications:
    1. The feature set does not contain strong non-linear patterns or feature
       interactions that predict weekend gaps.
    2. This is consistent with the Efficient Market Hypothesis: if a reliable
       non-linear gap signal existed in public technical/fundamental data,
       arbitrage would have eliminated it. The market prices in available
       information.

The one exception is fold 2022 (132 trees, AUC=0.593) — the highest AUC fold,
suggesting 2022's market conditions (high volatility, rate hikes) may have
temporarily made gap patterns more learnable.

10. RECALL BIAS — LIMITATION
--------------------------------------------------------------------------------
XGBoost exhibits a strong recall bias: Precision=0.544, Recall=0.723.
The model predicts GapUp far more often than GapDown.

The dataset has a slight GapUp majority (51.6%). XGBoost optimizes log loss by
default, which can be satisfied by always predicting the majority class when
signal is weak. The result is a high recall (catches most actual GapUps) but low
precision (many false GapUp predictions).

This inflates F1 (0.616 vs LogReg 0.567) without representing a genuine
improvement in discrimination. AUC-ROC is the correct comparison metric here
because it evaluates discrimination across all thresholds independently of the
majority-class bias.

Mitigation not applied: scale_pos_weight or threshold tuning could reduce this
bias, but given both models are in the same AUC range (0.56-0.57), addressing
the bias would not change the fundamental finding.

This is a theoretically coherent result, not a model failure.

LogReg outperforms XGBoost on COVID AUC (0.781 vs 0.724).

Interpretation caveat (same as LogReg): high COVID AUC is not evidence of
robustness. During the Feb-May 2020 crash, gaps were predominantly negative
(GapDown rate: 55%). A model that learns "most weeks end down" will achieve
elevated AUC not because it understands the crash mechanism, but because the
test distribution is directionally unidirectional. Both models' COVID AUC should
be interpreted as an artifact of the crash regime, not generalization ability.

XGBoost's lower COVID AUC vs LogReg suggests its non-linear decision boundary,
trained on normal market conditions, generalizes less well to the extreme
feature values during the COVID crash than the linear boundary of LogReg.

13. OVERALL INTERPRETATION & IMPLICATIONS
--------------------------------------------------------------------------------

The XGBoost analysis produces three findings relevant to the paper:

  1. Non-linear signal is absent or negligible
     The few-trees result (2-7 trees, validated by fixed n_estimators=200 check)
     is the most important finding. It means the relationship between our feature
     set and weekend gap direction is approximately linear — if it exists at all.
     XGBoost's gradient boosting machinery finds nothing beyond what the first
     shallow tree captures.

  2. LogReg is competitive or better
     In weak-signal, near-linear regimes, model simplicity is an advantage.
     Regularization in LogReg (L1, C=0.005-0.05) prevents overfitting to noise.
     XGBoost, despite early stopping, has more degrees of freedom and slightly
     overfits. AUC difference (-0.007) is within noise range but directionally
     consistent: simpler wins.

  3. Market efficiency interpretation
     Both models achieve AUC ~0.57 — modest but consistent improvement over
     random (0.50) and the majority-class baseline. This suggests a small,
     real, but exploitable-only-marginally signal exists in these features.
     The pattern is consistent with semi-strong market efficiency: prices
     reflect public information, leaving only marginal predictability.

Motivation for LSTM:
     XGBoost treats each Monday row as independent. If predictive signal exists
     in how features evolve across consecutive weeks (momentum building, RSI
     trending, OBV accumulating), XGBoost cannot detect it. LSTM is designed
     specifically to find temporal sequential patterns. The LSTM result will
     determine whether the residual signal (~0.57 AUC) is better explained by
     static feature values or by their sequential dynamics.

--------------------------------------------------------------------------------
15. LIMITATIONS
--------------------------------------------------------------------------------
  1. Recall bias: XGBoost predicts GapUp aggressively; F1 is misleading as
     a comparison metric. AUC is the correct metric for this analysis.
  2. Few trees: While validated as genuine (not early-stopping artifact), it
     limits XGBoost's expressive power and makes results close to a single
     decision tree rather than a true ensemble.
  3. No calibration: Predicted probabilities are not calibrated. If the model
     outputs 0.7 for a prediction, that does not necessarily correspond to a
     70% gap-up rate. Calibration curves were not generated.
  4. Pooled model: One model trained across all 25 tickers. Ticker-specific
     effects (e.g., MSFT vs WDAY gap behavior) are not modeled separately.
  5. Static rows: Each Monday is an independent observation. Sequential patterns
     across weeks are invisible to XGBoost — this is the LSTM's domain.
